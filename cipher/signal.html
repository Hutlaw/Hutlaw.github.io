Signal Explained

Diffie‐Hellman–like Key Exchange: Each party generates a key in JSON format. Using the “x” values of the sender’s and recipient’s keys a shared numeric value is derived that determines the Caesar cipher shift.

Caesar Cipher: The shared shift is applied to the message so that only someone with the correct key (which produces the same shift) can decode it.

This demo mimics the key agreement of Signal but uses a simple substitution cipher for demonstrative purposes.
